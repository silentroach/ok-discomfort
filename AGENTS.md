# ОК Дискомфорт

Есть группа поселков, которые обслуживаются компанией "ОК-Комфорт". При покупке участков в поселках
эта компания навязывается и отказаться от ее услуг крайне сложно. Работа компании непрозрачна - компания
не предоставляет никаких смет, только иногда абстрактные отчеты с тем, что она сделала (довольно скудные).

Недавно компания оповестила о существенном поднятии тарифов с 544 рублей за сотку до 815 рублей за сотку. И
если до этого все просто ворчали, но смирялись со стоимостью, то теперь это всех возмутило. Помимо поднятия
стоимости обслуживания, компания еще и выкатила на свой сайт новую редакцию договора, где собственник участка
сильно ограничивается в правах (еще сильнее, чем раньше), а ОК Комфорт наоборот может делать что хочет и когда
хочет.

В итоге собственники решили объединиться и привлечь юристов.

Репозиторий для ведения транскрипций совещаний и организации документации жителей коттеджных поселков, обслуживаемых компанией «ОК-Комфорт».

Содержит:

- Транскрипции звонков с юристами и экспертами
- Итоговые сводки и рекомендации
- GitLab Pages сайт для публикации материалов

# Структура проекта

## `calls/` — транскрипции звонков

Каждый звонок — отдельная папка с датой:

```
calls/YYYY-MM-DD/
├── README.md        # Контекст, участники, вопросы
├── PROCESS.md       # Использованные промпты (ссылки)
├── RESULT.md        # Итоговая сводка
├── raw/             # Исходные файлы транскрипции (asr.json, asr.vtt, asr.txt)
└── process/         # Промежуточные файлы обработки (.vtt)
```

## `docs/` — GitLab Pages (Jekyll)

Сайт на Jekyll с темой `pages-themes/dinky`:

```
docs/
├── _config.yml              # Конфигурация Jekyll
├── index.md                 # Главная страница с листингом постов
├── YYYY-MM-DD.md            # Посты с front matter
└── assets/css/style.scss    # Стили
```

### Формат поста:

```yaml
---
title: "Название"
date: YYYY-MM-DD
excerpt: Краткое описание
---
# YYYY.MM.DD - название

[Содержимое по структуре транскрипции ниже]
```

Посты отображаются на главной в обратном хронологическом порядке.

# Формат итоговых транскрипций звонков

## Структура документа

```
# YYYY.MM.DD - краткое название

## Цель
Описание цели встречи (2-3 предложения).

[Видео/аудио запись](URL)

## Участники

- <a id="p-id"></a>**Имя** — роль/организация (представление: 00:00:00)
- ...

## Вопросы

1. Вопрос 1
2. Вопрос 2
...

## Транскрипция по темам

### Тема 1

- 00:00:00 [Имя](#p-id) — Текст реплики
- 00:00:00 [Имя](#p-id) — Текст реплики
...

### Тема 2
...
```

## Правила оформления

### Заголовок

- Дата в формате `YYYY.MM.DD`
- Краткое описание темы

### Участники

- Каждому участнику присваивается уникальный `id` (латиница)
- Указывается имя, роль/организация, таймкод представления
- Добавляются ссылки на сайты/профили при наличии

### Вопросы

- Нумерованный список ключевых вопросов, обсужденных на встрече
- Формулировки берутся из повестки или перефразируются

### Транскрипция

- Группировка по темам/вопросам (использовать `###`)
- Каждая реплика: `ТАЙМКОД [Имя](#p-id) — Текст`
- Таймкод в формате `HH:MM:SS`
- Ссылки на якоря участников
- В случае, когда участник встречи только один, в транскрипции не нужно указывать его имя,
  а саму транскрипцию лучше разбивать на секции по темам + разбивать на параграфы (временная
  метка ставится у каждого параграфа)
- Если говорящий упоминает фразу "о чем я говорил выше/раньше" (или похожие по смыслу) и ты можешь найти 
  от него это обсуждение - сделай на него якорь, а с фразы сделай на него ссылку. То же самое, если говорят 
  про реплики других людей - если ты смог ее найти, то сделай так же ссылку.

## Требования к содержанию

- Сохранять правовую/фактическую точность высказываний — ничего нельзя выдумывать или пересказывать от себя
- Выделять конкретные рекомендации и договоренности
- Ссылки на документы/нормативные акты — сохранять
